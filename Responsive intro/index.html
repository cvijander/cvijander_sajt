Pravljenje responsive sajta

Da bi smo omogucili da nam sajt bude responsivan - tj da odgovara kako mobilnim telefonima, tabletima, tako i normalnim velicinama ekrana. Napravicemo u postojecem CSS -u kod koji shodno uslovu prepoznaje koja je velicina ekrana u pitanju i pokazuje kod tj prezentaciju koja je namenjena tom kodu. Dakle u CSS-u unosimo promenljivu @media(uslov){ i ovde unosimo koje zelimo da vrednosti se prikazuju razlicito }

Smartphone

Prilikom pravljenja sajta za smartphone koristimo uslov media(max-width:500px){..}

Tablet

Kod pravljenja tableta,znamo da je rezolucija veca od mobilnog telefona dok je manja od klasicnog ekrana i zato koristimo sledece uslove

@media(min-width:501px) and (max-width:768px) { ..}

Normal

Normal ili klasicna velicina ekrana tj pocetna rezolocija koja je koriscena na monitorima pocinje od 769 px i ide do 1200 px -takozvana velicina 1366*768(HD)

@media(min-width:769px) and (max-width:1200px) { .. }

Widescreen

Widescreen tj ekran visoke rezolucije @media (min-width:1201px) {.}

Landscape

Landscape je primer gde koristimo visinu kao uslov za postavku same rezolucije @media(max-height:500px ){.}

EM

Em je jedinica mere koja ide u odnosu na font-size roditelja.Tako npr u bodiju dajemo neku velicinu tipa 20px dok u paragrafu stavimo 2em sto je onda 40px tj velicina u odnosu na roditelja.Tako da se sve odnosi na roditelja, i ko je kako ugnjzden.

REM

Rem je jedinica mere koja se odnosi na root folder.Odnosi se samo na jednog roditelja.Tako sto se u CSS-u unosi html {font size:20px} i onda vrsimo poredjenje sa datom vrednoscu. Tj u odnosu na pocetnu vrednost ostalima dajemo,tako je npr 2rem ustvari 2 *20px itd.


Viewport height (vh)

Viewport height predstavlja vidno polje koje vidimo kada smo na sajtu, dakle ako stavimo da je vrednost height:100vh to zapravo predstavlja da je nas deo koda zauzeo celu visinu vidnog polja.


Viewport width (vw)

Viewport width kao sto ime govori je sirina vidnog polja. Dakle koliko je nesto siroko prilikom gledanja width:70vw oznacava da je nesto samo 70 posto sirine vidnog polja.


Pravljenje responsivnog sajta koristeci col

Kada zelimo da pravimo reponzivni sajt, mozemo se koristiti i floatom. Tj neophodno je da svaki artikal ili vec sekciju koju zelimo da koristimo ubacimo u novi div koja ima klasu col

i da taj sektor ili artikal ubacimo u tu klasu, zatim cemo u css vrsiti podesavanja vezano za col shodno velicinama ekrana.U css unosimo .col { float:left; cime omogucujemo da deo postane responzivan odnosno da prati levu stranu, tj da pluta.
Neophodno je uz float koristiti i clearfix tehniku..clearfix::before, .clearfix::after { display: table; content: " "; } .clearfix::after { clear: both; } pri cemu se klasa clearfi dodaje u sekciji iznad samog cola, tj kao njegov roditelj.

Sada u css delu unosimo tj pravimo media propertije shodno minimalnoj sirini ekrana i na osnovu toga vrsimo podele. Tako npr ako stavimo da je min-width:768px onda cemo dati uslov da je .col width:50%.

@media (min-width:768px) { .col { width: 50%; } @media (min-width:992px){ .col { width: 33.333%; } } @media (min-width:1200px) { .col { width: 25%; } }. Dakle rekli smo sledece ako je min sirina 768 onda je width 50 posto, sledeca velicina je 992px i width je 33,333 posto,dok najveca je 1200px i delimo sadrzaj na 25% tj cetvrtine.

nth-of-type (an+b)

Prilikom koriscenja floata i responziva , cesto cemo se sustretati da tekst nece da prodje tj da se postavi na pravo mesto, da bi smo to sigurno obezbedili koristicemo .col:nth-of-type(an+b) cime govorimo sledece. Ako zelimo da u rezoluciji tj gde smo rekli da je npr width bio 50 % onda je vrednost (2n+3), dok na sledecoj rezoluciji je (3n+4) (ako je nesto bilo 33.333% onda uzimamo prvi broj i stavljamo ga kao vrednost A , dok je vrednost B za jedan veci tj u ovom primeru 4, takodje moze se susresti kao i vrednost 1).
Procedura za izradu je sledeca u prvoj vrednosti postavljamo section .col:nth-of-type(2n+3){ clear: both; } dok tu istu vrednost u sledecoj rezoluciji proglasavamo clear:none i unosimo odnogovarajucu vrednost. section .col:nth-of-type(2n+3){ clear: none; } section .col:nth-of-type(3n+4){ clear: both; }Isti postupak ponavljamo i u vecoj tj sledecoj rezoluciji section .col:nth-of-type(2n+3){ clear: none; } section .col:nth-of-type(3n+4){ clear: none; } section .col:nth-of-type(4n+5) { clear: both; } Cime smo se osigurali da u jednoj rezoluciji odgovara samo jedna velicina dok su ostale ne upotrebljene.
